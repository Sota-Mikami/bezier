# Verify の UX — discovery（4ペルソナ）と方針提案

> **問い**: Implement 後の「Verify（検証）」体験が微妙。**コードを読まずに"正しい/完了"とどう確信するか**を4ペルソナに非誘導インタビューし、より良い・シームレスな体験方針を出す。
> **現状の Verify（DEC-050）**: ボタンを押すと AI が受入基準を読み直し、1つずつ **PASS/FAIL/BLOCKED/SKIP で自己採点**して一覧（verify.md）を出す。

---

## 1. インタビュー synthesis（Mai 一人maker / Kenji 作れないPM / Priya DSリード / Leo design engineer）

### 🔴 最大の発見（4人全員一致）— **AI の自己採点を、誰も信用していない**
現状 Verify の核心（AI が自分の実装を自分で採点）を、4人が明確に拒否:
- **Mai**:「AI の自己採点は確証バイアス。"できてほしいから pass" になる」
- **Kenji**:「実装した本人が自分のテストを書いて自分でパスしたのと同じ構造」
- **Priya**:「**信じません。以上。** 実装者とレビュアーを分離するのがガバナンスの基本」
- **Leo**:「それは peer review じゃなく **self-review**。チェックリストの"自己申告欄"」

→ **現 Verify は、全員が拒否した信頼モデルそのもの。「微妙」は表層でなく構造的。**

### 共通して欲しいもの（収束した4点）
1. **証拠 > テキスト合否**：全員「動いてる画面を見せて」。
   - before/after スクショ（**全員が筆頭で言及**）、実挙動、（Kenji/Leo）操作の**短い画面収録**、（Leo）fail の**再現コマンド＋ diff 該当行リンク**。
2. **検証は"別の何か"が担う**（AI は作る、正しさは実行環境/機械/自分の目）。
   - Leo「採点は AI でなく**実行環境**。AI=作る担当、正しいか=実行環境と俺」／ Priya「**DS そのものが採点官**になる」／ Mai「pass の根拠＝"このコンポーネントが render された"」。
3. **変更スコープ（blast radius）を可視化**。
   - Mai「変更ファイル多すぎ／指示してないファイルを触ってる＝不安」「`+3 files`、変更行数を出して」／ Leo「`git diff --stat` でスコープが頭のモデルと合うか先に見る」。
4. **受入基準の"構造"は価値、AI の verdict は無価値**（Leo）。「何を確認すべきかの構造化は要る、PASS/FAIL を正として受け取るのが問題」。

### ロール別の固有ニーズ
- **Mai（速度）**: 自動でいい＝lint/type/スクショ/変更ファイル一覧。**自分の目が要る＝auth・DBスキーマ・env を触った形跡**（"ここだけは見ないと不安、データ飛んだら終わり"）。「あとは自分で3秒触って OK」。
- **Kenji（説明責任）**: コードもデザインも読めない。要るのは①**自分のシナリオを AI が叩く画面収録** ②**"DS から外れていない"の一言**（自分で確認不能）③**ステークホルダーに説明できる1段落**。今ないのは②③＝「デザイナーが後で全部やり直しでは」という恐怖と経営説明責任。
- **Priya（監査）**: ①**DS token/component 使用の機械検証**（HEX 直書き＝最悪のサイン）②**shadow component 0 件の証跡** ③**どの DS 版数に準拠か** ④**承認者・タイムスタンプの不変ログ（override 証跡）** ⑤**回帰（既存を壊してない）**。＋ DPA / データ取扱い（IT・法務の契約テーブルに乗る前提）。
- **Leo（コード読める層）**: Verify は「コードを読めない層の安心装置」。自分には**構造（何を確認すべき）は要る／AI の採点は要らない**。理想＝**commit で影響 UI パスの Playwright スモークが自動実行 → パスごとに before/after＋pass/fail を diff と並べる**。orchestrator 文化では Verify＝各エージェントの「**完了報告の標準フォーマット**（テスト JSON / diff URL / スモークログ）」を人/オーケストレーターが集約。

### best / worst（デザイン参照）
- **best**: v0/Cursor の「生成→即プレビュー→コード読まず判断」（Mai/Kenji）。Cursor の inline diff **Accept/Reject キーボード一発**（Leo）。**DS Linter の bot コメントでマージブロック**（Priya）。
- **worst**: 情報ゼロの「LGTM」1行（Kenji）。「変更しました」→ マージ→無関係スタイル全上書き・ファイル数警告なし（Mai）。CI green だが snapshot が腐っていた（Priya）。CI fail/env 差分なのに "pass" 主張（Leo）。

## 2. 方針提案 — **「自己採点」をやめ、「証拠」を集めて一目で渡す**

> **原則**: Verify は AI の自己採点を廃止する。**AI は作る。検証は"別の何か"（実行環境・機械チェック・あなたの目）が担う。** 受入基準は「**何を確認するか**」の構造として残し、各基準に「**動いている証拠**」（before/after・実挙動・機械チェック結果）を紐づける。判定が機械化できない所は「**AI 判定（要目視）**」と正直にラベルする。

playbook 整合: Sierra/Anthropic の **Constellation of Models（"AI の問題の答えはより多くの AI"＝生成とは別の supervisor が検証）**。self-review でなく **独立した検証**。Bezier の楔（repo/DS 接地・repo-as-SoR）だからこそ、**実 repo/DS に対する機械チェック**を独自に出せる（＝Priya の言う"独立した採点官"を製品で提供）。

### 体験の作り直し（6手）
1. **verdict → evidence**：各受入基準に PASS/FAIL でなく**証拠**を出す。
   - 関連 UI の **before/after スクショ**（全員の筆頭ニーズ）、実挙動（可能なら短い操作録画）、FAIL は**再現（route/操作）＋ diff 該当行＋次の一手**。
2. **独立・自動・シームレスなチェック**（ボタンを押して待つ、をやめる）。turn 終了時に**実行環境が判定できるもの**を自動実行:
   - type/lint/build/（あれば）テスト、**変更スコープ**（changed files/lines＝blast radius）、**機微領域フラグ**（auth/db/env を触った？ ← Mai）、**DS 準拠**（使用 token/component がカタログに在る・shadow/HEX 直書きなし ← Priya）。**AI の言葉に依存しない**。
3. **AI の役割＝判事でなく"証拠の収集・提示"**：エージェントは「チェックを走らせ、before/after を撮り、再現を書く」。機械判定でない所だけ「AI 判定（要目視）」と明示。可能なら**生成とは別のエージェント（supervisor）**が検証（Constellation）。
4. **人の一目＋sign-off**：非コーダーが5秒で読める **confidence ビュー**＝ ✅機械検証済（信頼可）/ 👁あなたの目が要る（機微領域・Mai）/ プレーンな**"何が変わったか"1段落**（Kenji の説明責任）。Accept は**キーボード一発**（Leo）。
5. **配置＝Implement のレビューに織り込む**（"埋もれたサブタブ＋手動ボタン"をやめる）：turn 終了で **証拠パネルが自動で出る** → **レビュー＝検証**になる。これが CEO の言う"シームレス"。
6. **（enterprise/Priya）監査証跡**：証拠＋チェック＋承認者＋タイムスタンプを**不変ログ**で（後段。repo-as-SoR と直結）。

### 各ペルソナの充足
| 要求 | 手 |
|---|---|
| Mai: 速く・機微だけ自分の目 | ②機微フラグ＋②スコープ＋①before/after、残りは Accept |
| Kenji: 説明できる・DS逸脱の一言 | ①画面録画＋②DS準拠チェック＋④1段落 |
| Priya: 独立検証・DS準拠・回帰・監査 | ②機械DSチェック＋③supervisor＋⑥不変ログ |
| Leo: 実行環境が採点・再現・キーボード | ②自動スモーク＋①再現/diff リンク＋④Accept |

---

## 3. 段階導入（安価×高価値から）
1. **before/after スクショ＋変更スコープ（files/lines）＋機微領域フラグ**を turn 終了で自動提示（①②の一部・全員効く・最小工数）。verdict を"証拠"に置換。
2. **confidence ビュー**（✅機械検証済 / 👁要目視 / "何が変わったか"1段落）＋ Accept キーボード（④）。
3. **DS 準拠＋回帰の機械チェック**（②の核・Bezier の差別化・Priya/Kenji）。
4. **独立 supervisor エージェントで検証**（self-review 廃止・Constellation）。
5. **不変な監査ログ**（enterprise）。

---

## 4. 一言
> 今の Verify は「**AI が自分の宿題に自分で丸をつけ、その丸を一覧で見せる**」。4人全員がその丸を信じていない。→ **「丸」をやめ、"動いてる証拠"（before/after・実挙動・機械チェック）を turn 終了でシームレスに差し出し、機械が決められる所は機械が、機微な所はあなたが、判事を AI から外す。** 受入基準は"何を見るか"の地図として残す。

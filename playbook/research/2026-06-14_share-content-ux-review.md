# 「共有する内容」UX レビュー — 専門家＋ペルソナ3名

CEO「『共有する内容』のトグル（アプリ/Spec/デザイン/実装）、それぞれ何のことかわからない。徹底的に磨きたい」。Principal Designer＋persona（Mai/Kenji/Tom）に並行 dispatch。

## 収束点（4体の共通項）

1. **ラベルが内部用語＝受け手目線がない**。「何が共有されるか」が伝わらない。
2. **「実装」が最悪**。Mai「一番怖い、間違えて送りそう」/ Kenji「上司に GitHub リンク？絶対送らない、不信感」/ Tom「論外、工数が透けて単価交渉のネタ」。→ **開発者向けと明示**して既定 OFF。
3. **「Spec」も非エンジニアに通じない**。Kenji の上司は「スペック表？」。
4. **1行の説明が要る**（information scent）。ピルは説明を持てない＝そもそもこの不満の原因。
5. **既定 全 ON が怖い**（Mai/Kenji）。「実装まで送ってた」やらかし。
6. **送る前に確認したい**（4体全員）。「経営会議前日に実装入ってた、は死ぬ」。
7. **プリセットが欲しい**（Tom）。「クライアント用＝アプリ＋デザイン」「ハンドオフ用＝Spec＋実装」をブランドごとに。
8. **誰向けかで二分**される：見せる成果物（アプリ・デザイン）vs 内部/開発（Spec・実装）。

## 確定ラベル＋説明（Principal Designer 本命、persona で補正）

| key | 新ラベル | 1行説明 | 既定 |
|---|---|---|---|
| app | **動くアプリ** | 受け手がその場で触って試せる（Web に公開） | ON |
| design | **採用したデザイン** | 実装前に選んだ画面デザイン | ON |
| spec | **企画・要件** | なぜ作るか・何を実現するか | OFF |
| impl | **コードと変更履歴** | 開発者向け：GitHub リンクとコミット一覧 | OFF |

## レイアウト：ピル → 説明付き行（2グループ）

CEO は先にピルを選んだが、不満の核＝「各項目が何か分からない」の解は **1行説明**。ピルは説明を載せられないので **説明付きの行**に変える（情報量は増えるがクロームではなく明確化。design-system 原則1に抵触しない＝設定パネル内）。

```
共有する内容を選んでください

  見せる成果物
  ┌─────────────────────────────────┐
  │ ✓  動くアプリ                    │  ON（薄い背景＋✓）
  │    受け手がその場で触って試せる   │
  ├─────────────────────────────────┤
  │ ✓  採用したデザイン              │  ON
  │    実装前に選んだ画面デザイン     │
  └─────────────────────────────────┘

  背景・記録（任意）
  ┌─────────────────────────────────┐
  │    企画・要件                    │  OFF
  │    なぜ作るか・何を実現するか     │
  ├─────────────────────────────────┤
  │    コードと変更履歴              │  OFF
  │    開発者向け：GitHub と変更の記録 │
  └─────────────────────────────────┘

  [ 共有する → ]
  完了 → 「開いて確認」してから URL を送る、を明示
```

グルーピングそのものが「上＝受け手に見せる／下＝任意の内部記録」を伝え、Kenji/Tom の「誰向け？」を解消する＝**軽量プリセットの代替**。

## 既定 ON/OFF
- **動くアプリ・採用したデザイン＝ON**（成果物。コード/内部を含まないので安全）。
- **企画・要件・コードと変更履歴＝OFF**（特に「コードと変更履歴」は常に既定 OFF）。
- 既存の `journeyLayers`（localStorage）に保存。初回未設定のみこの既定。

## backlog（この磨き込みの次）
- **F-7 プリセット**：「クライアント用／ハンドオフ用」を保存・ワンタップ適用（Tom 強要望、複数ブランド対応）。
- **F-8 送る前プレビュー**：生成後の「開く」を“送る前の確認”として動線上明示（最小）。専用プレビュー画面は後。
- **F-9 存在しない層はトグルを出さない**：デザイン未採用／PR なし等は該当トグルを非表示or無効（空セクション共有の防止）。

# v0.5 slice 2 — 実装ループ（Spec → worktree → AI 実装 → diff → merge）実装プラン

> dogfood で CEO が「Spec を書いても行き先がない（branch/worktree で AI に実装させたい）」と詰まった所を解消。continuum の核「**意図 → AI が実行 → 記録**」を一周させる。
> 前提: slice 1（起票＋Spec）済。DEC-011（Design slot 廃止 / Decision 自動）を本 slice で反映。

## 0. このスライスのゴール
**Issue の Spec から、branch+worktree を切り、AI（claude/codex）に実装させ、diff を見て採用/却下し、merge すると Decision が自動で残る。**

## 1. 先に slot スリム化（DEC-011）
- Issue 詳細から **Design slot を削除**（`createSlot` の design 分岐・UI カード除去）。
- **Decision を手書き slot から外す**（`+Decision` 導線削除）→ merge 時に自動生成（§4）。
- Issue 詳細の右ペインは **Spec 中心**＋下記「Changes / 実装」。

## 2. Rust git コマンド（`src-tauri/src/lib.rs` に追加）
`std::process::Command` で git CLI を叩く（最小・堅牢）。すべて `reject_traversal` で安全確認。
- `git_is_repo(path) -> bool`
- `git_worktree_add(repo, branch, worktree_path)`（無ければ branch 作成して worktree 追加）
- `git_diff(worktree_path) -> String`（`git -C <wt> diff` のテキスト。未コミット変更＝AI が触った差分）
- `git_status(worktree_path) -> String`（変更ファイル一覧 `git -C <wt> status --porcelain`）
- `git_commit_all(worktree_path, message)`（`git -C <wt> add -A && commit -m`）
- `git_worktree_remove(repo, worktree_path)`（破棄。`git worktree remove --force` + branch 削除は任意）
- （merge は slice 2 では「branch に commit を残す」までを基本とし、main への merge は任意ボタン `git_merge(repo, branch)` を用意するが**既定は PR 運用＝branch に置くだけ**。DEC-008/G1' 準拠）
- Tauri の `invoke_handler` に登録。

## 3. データ層 `src/lib/git.ts`（新規）＋ `issues.ts` 拡張
- `git.ts`: 上記コマンドの薄いラッパ。
- worktree 置き場 = `.continuum/worktrees/<ulid>/`（gitignore 済）。`.continuum/issues/<ulid>/worktree.json` に branch/path/baseSHA を記録（揮発）。
- branch 規約 = `issue/<ulid>-<slug>`（DEC-009 G2）。
- `issues.ts`: status を open→in-progress（worktree 作成時）→merged（commit/merge 時）に更新。slice 2 では status は issue.md frontmatter のままで可（slice では local 正本の厳密分離は後回し）。

## 4. UI — Issue 詳細に「実装」セクション
1. **「Implement with AI」ボタン**（Spec が在る Issue で活性）:
   - `git_is_repo(root)` 確認 → `issue/<ulid>-slug` branch + `.continuum/worktrees/<ulid>/` worktree 作成 → status=in-progress。
   - **handoff を生成**（issue.md＋spec.md＋「この worktree で実装して」指示）→ 既存 `agent-launcher`/`pty` 機構で **claude/codex を worktree cwd で起動**（v0.2 流用、cwd=worktree）。
2. **Changes ビュー**: `git_diff(worktree)` を読んで **diff を表示**（追加/削除を色分け・read-only で十分）。`git_status` で変更ファイル一覧。ポーリング or 手動更新ボタン。
3. **採用 / 却下**:
   - 採用 → `git_commit_all(worktree, "<issue title>")`（branch に commit）→ status=merged → **Decision 自動 draft（§5）**。（main への merge ボタンは任意・既定は branch+PR 運用）
   - 却下 → `git_worktree_remove` ＋ branch 削除 → status=open に戻す/破棄。

## 5. Decision 自動生成（DEC-011）
- 採用時、`decision.md` を **自動 draft**: title=issue / 文脈=spec「なぜ」/ 決定=spec「何を」/ 影響=`git_status` の変更パス＋issue.screens / 関連=branch・spec。ADR 形式（§3.5）。
- `.continuum/drafts/<ulid>-<slug>/decision.md` に書き、Decisions ビューに出る。**人が PR 上で確認・修正**前提（slice 2 では生成まで）。

## 6. 受け入れ（dogfood）
1. git repo を開いた状態で Issue に Spec を書く → 「Implement with AI」→ worktree+branch ができ、AI がターミナルで実装を始める
2. **Changes に diff が出る** → 採用 → branch に commit ＋ decision.md 自動生成 → Decisions に出る
3. 却下で worktree/branch が消え、元に戻る
4. 非 git フォルダでは「Implement」は無効＋理由表示

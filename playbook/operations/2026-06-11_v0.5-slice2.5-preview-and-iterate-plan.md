# v0.5 slice 2.5 — Preview（視覚レビュー）＋ 反復 re-implement 実装プラン

> DEC-012: ループは一方通行でなく反復。コード diff でなく **実物（worktree のアプリ）を iframe で見て**、仕様を磨いて再実装する review↔refine サイクルにする。
> 前提: slice 2（worktree＋AI実装＋diff＋accept）済。

## A. Preview — worktree のアプリを起動して iframe 表示
1. **dev コマンド/ポートの解決**:
   - repo の `package.json` `scripts.dev` を**自動検出**して既定に（無ければ空＝手入力促す）。
   - `<root>/.bezier/config.json`（gitignore 配下）に `{ devCommand, port }` を持ち、**設定欄で上書き可**。port は未指定なら空きポートを採番（例: 4100+）。
2. **起動**: Issue review で **「Preview」** → 既存 pty 機構で worktree(cwd)で `<devCommand>`（必要なら `-p <port>` 付与）を起動。ready 待ち（`/` が 200 になるまでポーリング、タイムアウト＋ログ表示）。
3. **表示**: `http://localhost:<port>` を **iframe**（既存 `screen-frame.tsx` / canvas の iframe 描画を流用）で Issue review に表示。
4. **text diff は残す**（精度・エンジニア手渡し用）。タブ/トグルで「Preview ⇆ Diff」。
5. **停止/後始末**: review を離れる/Issue を閉じる時に dev server プロセスを kill（pty 破棄）。worktree 破棄時も kill。
6. 注意: **素の Web プロダクト向け**前提（Tauri デスクトップアプリ自体の iframe は対象外）。起動失敗・port 衝突は明示エラー。

## B. 反復 re-implement（review↔refine サイクル）
- **worktree/branch は accept まで持続**（slice 2 で既に残置）。
- **Spec を編集 → 「Re-run AI」**: 更新後の spec.md ＋「前回からの変更/直してほしい点」を handoff に載せ、**同じ worktree で agent を再起動 or 継続**（埋め込みターミナルのセッションは生きているので、追加指示を送る／新規 run のどちらか。まずは "Re-run = 新しい handoff を投げる" でよい）。
- サイクル: **Spec 磨く → Re-run → Preview → diff 確認 → また磨く → … → Accept**。
- Accept 時に Decision 自動生成（slice 2 のまま）。spec・code・decision が共進化。
- UI: Issue review に「Implement / Re-run」「Preview」「Diff」「Accept / Discard」を**サイクルとして**並べる（ウォーターフォールのステップでなく、回せる操作群に）。

## C. 非スコープ（slice 3）
- **AI-assisted spec ＋ 画面参照（2-1）**: spec エディタ内の LLM チャットで仕様を共同検討・既存画面を参照しながら記述。
- main 自動 merge / gh PR / drift harness（slice 4）。

## D. 受け入れ（dogfood）
1. Web repo を開き Issue→Spec→Implement→（AI 実装）→ **Preview で実物が iframe 表示される**
2. **Spec を直して Re-run → Preview が更新**される（反復できる）
3. Diff と Preview を行き来できる／Accept で commit＋Decision／Discard で後始末
4. dev コマンド自動検出＋設定上書きが効く／起動失敗時に分かる

## E. 検証ゲート
`npx tsc --noEmit`=0 ／`npx eslint`=clean ／`npx next build` 成功 ／（pty/iframe 追加で）dev(3210)`✓ Compiled`＋`/issues`200。実起動・iframe は CEO dogfood ゲート。

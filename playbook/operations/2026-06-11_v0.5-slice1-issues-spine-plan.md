# v0.5 slice 1 — Issues 背骨（pre-git・ローカル）実装プラン

> IDE 風 workspace を **Issue で回るアプリ**へ。要件 = `product/specs/2026-06-11_ia-and-issue-model.md`（§1 メニュー / §3 Issue モデル）。
> **slice 1 スコープ**: Issue の作成・一覧・詳細（spec/design/decision を CM エディタで編集）＋ Decisions 集約。**git/worktree・注釈・Product Board・drift harness は対象外**（slice 2-4）。

## 0. slice 1 の割り切り（pre-git）
- git/worktree がまだ無いので、**全 issue は `.bezier/`（gitignore・ローカル）に置く**。durable の `docs/issues/` 昇格 & PR は **slice 2** で。
- status は slice 1 では issue.md frontmatter に持つ（全部ローカルゆえ main 汚染の懸念なし）。slice 2 で `.bezier/issues/<ulid>/status` に分離。

## 1. データモデル（slice 1）
```
.bezier/drafts/<ulid>-<slug>/
├── issue.md       frontmatter: id(ULID) / title / status / screens? / labels? / created。本文=説明
├── spec.md        （+ で作成・§3.5 テンプレ）
├── design/        （+ で作成・notes.md を生成）
└── decision.md    （+ で作成・ADR テンプレ）
```
- すべて `.bezier/`（既に gitignore 済）。`writeFile`（Rust）は親フォルダを自動作成するので mkdir コマンド不要。
- **ULID**: `ulid` npm を追加（小さい）。slug = title を kebab 化（空なら `untitled`）。

## 2. データ層 `src/lib/issues.ts`（新規）
- 型: `IssueStatus = "open" | "in-progress" | "merged"`、`Issue = { id, slug, dir, title, status, screens?, labels?, created, slots: { spec: boolean; design: boolean; decision: boolean } }`。
- `listIssues(root)`: `.bezier/drafts/*/issue.md` を走査 → frontmatter parse（`splitFrontmatter` 再利用）→ slot 存在確認（listDir）→ created desc ソート。
- `createIssue(root, title)`: ULID 採番 → `.bezier/drafts/<ulid>-<slug>/issue.md` を frontmatter+stub 本文で writeFile → Issue 返す。
- `issueDir(root, id)` / `readIssue(root, id)`（issue.md + slot 存在）。
- `createSlot(root, issue, "spec"|"design"|"decision")`: 規約パスにテンプレを writeFile（spec.md=§3.5 / decision.md=ADR / design/notes.md）。既存なら何もしない。
- `updateIssueMeta(root, issue, patch)`: issue.md frontmatter 更新（title/status/labels）。本文（説明）保持。
- 注意: 既存の `markdown.ts splitFrontmatter` を parse に使う（serialize は frontmatter を yaml で emit、本文はそのまま）。

## 3. UI
### 3.1 ナビ（`app-sidebar.tsx` 刷新）
- レガシー項目（Inbox/Design Issues/Components/Workspace）を **`Product / Issues / Decisions / Repo`** に置換。
  - **Issues**（新規・本命） / **Decisions**（集約ビュー） / **Product**（placeholder「coming」） / **Repo**（＝現 `/workspace` IDE：folder-open + file tree + Canvas + terminal を残す）
- ルーティング: `/issues`（一覧）、`/issues/[id]`（詳細）、`/decisions`、`/workspace`（既存=Repo）、`/product`（placeholder）。ランディングは `/issues`。
- Agent は引き続き常駐（Repo 側のターミナル/handoff はそのまま）。slice 1 では Issue 詳細からの委譲は未配線でよい。

### 3.2 Issues 一覧 `/issues`
- 「New issue」ボタン（title 入力 → `createIssue` → 詳細へ）。
- カード/行: title・status バッジ・slot チップ（spec/design/decision の有無）・created。クリックで詳細。

### 3.3 Issue 詳細 `/issues/[id]`
- ヘッダ: title（インライン編集）・**status ドロップダウン**（open/in-progress/merged）・labels。
- **二ペイン**（§3.3）:
  - 左 = **thread（最小）**: 「起票（created）」＋ issue.md 本文（説明）を表示。本文編集は CM エディタ（小）or 省略可。slice 1 は最小でよい。
  - 右 = **artifact slots**: `Spec` / `Design` / `Decision` カード。
    - 在る → クリックで右ペインに **MarkdownEditor（CM6・既存 `Editor`/`MarkdownEditor` を再利用）** をマウントして編集・保存（save IF 既存）。
    - 無い → `+ Spec` / `+ Design` / `+ Decision` で `createSlot` → 生成 → 編集へ。
- **CM エディタ再利用**: 既存 `workspace/page.tsx` の `Editor` パターン（readDoc → MarkdownEditor + Save + reload）を Issue 詳細でも使えるよう、slot ファイルの絶対パスを渡して同じ編集体験にする。

### 3.4 Decisions `/decisions`
- 全 `.bezier/drafts/*/decision.md` を集約して時系列リスト（title/issue/decided）。クリックで該当 decision を CM 表示。読み中心。

## 4. 受け入れ（dogfood で確認）
1. 「New issue」で issue が作れ、`.bezier/drafts/<ulid>-<slug>/issue.md` が生成される
2. 詳細で `+ Spec` → spec.md 生成 → CM エディタで編集・保存できる（zero-diff も維持）
3. 一覧に出る・status を変えられる・Decisions に decision.md が集約される
4. 既存 Repo（/workspace の編集/Canvas/terminal）は壊れず動く

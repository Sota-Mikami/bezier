## 採用方式

**汎用プレビュージェネレータ `spike/generate-preview.mjs`** を新規実装し、任意の Next.js app-router repo に対して scene-graph から preview ルートを動的生成する方式を採用した。

ISSUE-003 の chom-chom 専用手書き page.tsx を完全廃棄し、同一コードで複数 repo を処理するハーネスに昇格させた。

### 実装構成

```
spike/
  generate-preview.mjs    汎用 preview ジェネレータ（新規）
  screenshot-generic.mjs  汎用スクリーンショット (indexName + baseUrl 引数)(新規)
  cli.mjs                 preview サブコマンド全面改修（gen-preview + dev server 自動起動）
  out/
    render-chomchom.png     chom-chom preview (汎用ハーネス版)
    render-template.png     template preview (汎用ハーネス版) ← 新規 repo 実描画
    preview-summary-{chomchom,template}.json
```

### ハーネス設計（汎用化の核心）

```
gen-*.json + index.json → generate-preview.mjs
  1. framework 検出 (Next.js app-router / vite / unknown)
  2. root layout.tsx から provider 自動検出（AIContextProvider / CommentContextProvider など）
  3. 各ノードの render 戦略決定:
     - existing_component → index の実ファイルパスを @/ alias で import 解決
       + props は prop名 heuristics でモック合成
       + ErrorBoundary でラップ（runtime error を FALLBACK に落とす）
     - generated / primitive → scene-graph の role/title/note から inline JSX 生成
     - AuthGate / PasswordGate / CommentLayer / Session → FALLBACK (blocking-provider)
     - screen-kind component → FALLBACK (full page, too complex)
  4. standalone layout.tsx 生成（AuthGate バイパス + 安全なプロバイダーのみ wrap）
  5. gitignore に continuum-preview/ を追記
```

```
cli.mjs preview <indexName> [--port <n>]
  Step 1: generate (intent 省略時スキップ)
  Step 2: gen-preview (generate-preview.mjs 呼び出し)
  Step 3: dev server 自動起動（ポートが応答しなければ npm run dev 起動・最大30秒待機）
  Step 4: screenshot (screenshot-generic.mjs)
  → dev server を使い終わったら SIGTERM で停止
```

## repo 固有の壁（明記）

### chom-chom の壁: AuthGate による全ページ認証要求

chom-chom の root `app/layout.tsx` が `<AuthGate>` で children を wrap している。Next.js App Router では segment layout（`continuum-preview/layout.tsx`）は root layout を上書きしない（ネストする）ため、`/continuum-preview` にアクセスしても root の AuthGate が適用されてログイン要求が表示される。

**解決に必要な変更**: `src/components/AuthGate.tsx` の先頭に `usePathname()` チェックを追加し `/continuum-preview` を bypass するコード 3行。これは ISSUE-003 で手動実施済み（gitignore 管理外で現在は不在）。

**制約との関係**: ISSUE-004 制約「既存ファイル不変」の下では自動注入は NG（auto-mode classifier が拒否）。

**選択肢**:
1. CEO が手動で AuthGate.tsx の bypass を 1行追加する（3行の最小変更）
2. `generate-preview.mjs` に "--allow-gate-bypass" フラグを追加し明示的に許可するワークフロー
3. continuum 専用の env var `CONTINUUM_PREVIEW=1` で AuthGate が bypass するよう設計変更（本格実装）

### alloy の壁: repo がディスク上に存在しない

`out/alloy.json` の `repo` フィールド `/Users/sota/Workspaces/Personal/projects/alloy` が存在しない。`generate-preview.mjs` は exit 2 で正直に報告し、cli は「repo が存在しません」と出力してスキップする。

### template の壁: ScreenPanel の complex ref props

`ScreenPanel` の props 型 `Props = { screen: Screen | null; onClose: () => void }` はcomplex object 型。index の `props.kind === "ref"` かつ ref 名が "Props" のため、ジェネレータが FALLBACK として扱う。

template 自体の認証はコメント機能（Liveblocks）専用で、プロトタイプ本体はauth-free。そのため `/continuum-preview` は認証なしで 200 を返してプレビューが表示される（chom-chom との重要な違い）。

## 次の論点（ISSUE-005 候補）

1. **AuthGate bypass の標準化**: `continuum preview` が安全に動くための env var フラグ（`CONTINUUM_PREVIEW=1`）を提案し、各 repo オーナーが自発的に対応できる設計にする

2. **alloy のローカル clone**: alloy が既存 index/gen を持っているがディスク上にない。clone して `preview alloy` が動く状態にするか、`extract` + `generate` を別 repo で再取得

3. **generated node の質向上**: 現状は role + title のみ。scene-graph に fields（フォームフィールド配列等）が含まれていれば、実際のフォームUIを生成できる

4. **chom-chom VocabFlashcard の freeplay モック**: `freeplay={true}` が型不一致でErrorBoundaryに落ちている。`freeplay={{ entries: [] }}` を正しく渡せれば「空のフリープレイ」として描画できる

5. **app/ canvas 統合**: preview ルートを continuum app の canvas に統合し、scene-graph 編集 → リアルタイムリレンダリングのUXループを実現

# Bezier LP — First View & スクロールインタラクション 5案

> ブリーフ（`2026-06-13_LP-design-brief-for-LLM.md`）への回答。
> 全案共通の制約: モノクロ（ink #1c1c24 / paper #f7f6f3 / dark #0e0e12・lit-black可）／ペンツール文法（四角アンカー・丸ハンドル・接線・ベジェ曲線・構成グリッド）／イージングは必ず cubic-bezier／黒い画面で脅さない。

---

## 案1 — **Atelier（設計中の言葉）** ★推奨・実装済
**一行**: ベタ塗りの thesis がスクロールで「ペンツールの中」に分解され、巨大な一本のベジェがエージェントの手で描き直され、最後に言葉が再びインクで完成する——**3幕構成のタイポグラフィ**。

```
┌──────────────────────────────────────────────┐
│  ◇ Bezier      Docs            GitHub [WL]   │
│ ┄┄┄┄┄┄┄┄┄ faint construction grid ┄┄┄┄┄┄┄┄┄ │
│      ╭─────────────────────────╮○←handle     │
│  ◇──╯  Hold the handles   ←────╯             │  ← 巨大スイープが言葉の背後を通る
│        （ベタ塗り 800wt）                      │
│   [tagline badge]  JPサブコピー                │
│   [ email ][ ウェイトリストに参加 → ]          │
└──────────────────────────────────────────────┘
```

**スクロール（pin 240vh・スクラブ）**
- **第1幕 0–12%**: ベタ塗りの言葉＋背後に lit-black の巨大スイープ（ロード時に dash で自動描画・両端に◇アンカー、接線の先に○ハンドル）。グリッドはかすかに。
- **第2幕 12–62%**: インクが薄れ→**アウトラインが左から描かれ**→輪郭に沿って**四角アンカーが密に出現**、3個に1個は**接線＋丸ハンドル**を伸ばす（Orvior）。グリッドが濃くなる＝製図室。
- **第3幕 58–96%**: **エージェントが巨大スイープの2つの丸ハンドルを掴んで動かす**（強調リングが灯る）→ 曲線が S を反転して**描き直される**。同時に言葉のアンカーが退き、**インクが再充填**＝完成。グリッドが引いて、整った1枚のポスターで終わる。
- イージング: 各レイヤー smoothstep スクラブ（可逆）。ハンドル移動だけ `cubic-bezier(0.22,1,0.36,1)` の質感。

**on-thesis**: 「言葉（プロダクト）は曲線でできている→ハンドルは見える→**動かすのはエージェント**→完成」。文法の全部品を使い、因果（ハンドル→曲線）を画面の主役にする。
**実装**: Canvas 2D 1枚。テキストをオフスクリーンでラスタ→輪郭点抽出（実装済の手法）。フォント不要・依存ゼロ・static export OK・reduced-motion は完成フレーム。
**リスク**: 中間状態がノイズに見える→ 間引き距離・アンカーの白抜き・左→右ステガで「意図した製図」に保つ。

## 案2 — **One Line（一本の線がページを描く）**
**一行**: ヒーローで生まれた**一本の太いベジェ**がスクロールでページを下り、各セクションの図解・枠・下線を**通りすがりに全部描いていく**。LP 全体が「エージェントの一筆書き」。

```
[Hero]   ◇──○        ╭───────╮
         Hold the handles     │
                              ↓ scroll
[Beats]  ╰→ ①の囲み枠を描く → ②の矢印になる
[Feat]   ╰→ カードの下線 → デモ部品の輪郭
[CTA]    ╰→ 最後にロゴの曲線そのものに収束 ◇──○
```
**スクロール**: ページ全長の SVG パス1本を `stroke-dashoffset` でスクラブ。ペン先（●）が常に viewport 中央付近。
**on-thesis**: 「描く主体は一人（エージェント）、あなたは要所のハンドルだけ」が構造で伝わる。最後がロゴに収束するのが回収。
**実装**: 全長 SVG＋section ごとの座標合わせ。**レイアウト変更に弱い**（レスポンシブの座標管理が高コスト）。
**リスク**: スクロールジャックに見える・モバイルで破綻しやすい。LP が安定してから上級編として。

## 案3 — **Grip（握れるヒーロー）**
**一行**: ヒーロー＝**実際にドラッグできる**ポスター大のベジェエディタ。あなたが○を握って曲線を変えられる。スクロールするとエージェントが引き継いで描き切る。

```
│   Hold the handles                │
│   ◇━━━━━━━━○ ←ドラッグ可（吸い付く）│
│      ╲＿＿＿ 曲線がライブ再描画 ＿╱ │
│   [CTA]                           │
```
**スクロール**: 0–30% ユーザー操作ゾーン（触らなければ idle で誘うパルス）→ 30%〜 ハンドルが自走（=エージェント）し、曲線ファミリーが増殖して**ワイヤーフレームの形に整列**→ 完成。
**on-thesis**: 「Hold the handles」を**文字通り体験**させる唯一の案。触った人は忘れない。
**実装**: Canvas + pointer events。難度中。タッチで縦スクロールと競合するため、ドラッグ領域の設計が肝。
**リスク**: 触られなければただの線。idle アニメで誘導必須。おもちゃ化の一歩手前を狙う統制が要る。

## 案4 — **Blueprint（構成図のポスター）**
**一行**: ファーストビュー全面が**バウハウスの製図ポスター**——巨大な円・四角・対角線・寸法線・mono 注記で「Bezier」を組版。スクロールでガイドが消えインクが満ち、**仕様書→完成品**に解像していく。

```
┌────────────────────────────┐
│ ┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼┼ │
│ ┼  ◯ 64px      ◇──────○  ┼ │
│ ┼   ╲   B E Z I E R      ┼ │ ← 寸法線・注記つき巨大組版
│ ┼    ╲__╱    r=12  ┼┼┼┼  ┼ │
└────────────────────────────┘
```
**スクロール**: ガイド/寸法/グリッドの opacity が段階的に 0 へ、図形は outline→lit-black 塗りへ、文字は構築線→確定ウェイトへ。「spec が描かれて製品になる」。
**on-thesis**: Spec→Design→実装 というプロダクトのループそのものをポスター1枚で。
**実装**: レイヤー分けした SVG の opacity/fill スクラブ。簡単で頑丈。ただし**動きの山場が弱い**（消える/満ちるだけ）。
**リスク**: 「綺麗な静的ポスター」止まり。山場として1要素だけ大きく動かす補強が要る。

---

## 案5 — **Swarm（無数の判断が、一つの形になる）**
**一行**: tryglen 級の存在感——**数百個の「◇+○ペア（ロゴの原子）」の群れ**が画面下中央で穏やかに泳ぎ、スクロールで見えない曲線に吸着・整列し、**1つの巨大なロゴマーク**へ凝縮する。

```
│        Hold the handles          │
│            [CTA]                 │
│      ・◇○ ◇・○。◇ ○・◇。      │
│    ◇○・。◇ ○ ◇・○ 。◇○・     │ ← 高密度の群れ（呼吸・うねり）
│      。◇・○ ◇。○・◇ ○        │
```
**スクロール**: 0% 自由遊泳（perlin 流）→ 40% 曲線族に吸着、接線が繋がり始める → 80% 全員が**ロゴの座標**へ snap、最後の1本の曲線が dash で描かれて完成。
**on-thesis**: 「無数の小さな判断（制御点）→1つの描かれたプロダクト」。tryglen の密度ベンチに最も正面から応える。
**実装**: Canvas 2D パーティクル＋ターゲットモーフ（ロゴパスから座標サンプル）。難度中。60fps には数を 400–600 に制御。
**リスク**: 粒が小さいと「ノイズ」、大きいと「うるさい」。粒密度と速度の調律に時間がかかる。

## 比較

| 案 | 存在感 | 独自性(ownable) | 実装コスト | モーションの craft | リスク |
|---|---|---|---|---|---|
| 1 Atelier | ◎（タイポ＋巨大スイープ） | ◎ 文法フル活用 | **低**（実装済基盤） | ◎ 3幕の物語 | 低 |
| 2 One Line | ○ | ◎ | **高**（座標管理） | ◎ | 高（responsive） |
| 3 Grip | ○（触れば◎） | ◎ 唯一の体験型 | 中 | ○ | 中（誘導設計） |
| 4 Blueprint | ◎（静的） | ○ | 低 | △ 山場不足 | 中（静的止まり） |
| 5 Swarm | ◎◎（ベンチ直球） | ○ | 中 | ○ | 中（調律工数） |

## 推奨
**今すぐ＝案1 Atelier を完成形に磨く**（基盤実装済・文法の物語性が最強・リスク最小）。**次ステップ＝案5 Swarm** を第2セクション or 将来のヒーロー差し替え候補として試作（tryglen 級の密度はこれが最短）。案3 の「ドラッグできるハンドル」は**Atelier の巨大スイープに後付け**できる（山場の前にユーザーが触れる→エージェントが引き継ぐ、で両取り）。

## ページ全体への展開（モーション言語の統一）
- 各セクション見出しの下に**「描かれる罫線」**（◇→曲線が dash で描画→○、IntersectionObserver 発火）＝ヒーローの文法を 1/100 スケールで反復。
- 箇条書きの bullet を丸でなく**四角アンカー**に。
- Features カードの hover で、アイコンに小さな接線＋ハンドルが 150ms で伸びる。
- 最終 CTA 上で、ヒーローの巨大スイープを**再登場**させて閉じる（ページ全体が1つの作画セッションだった、という回収）。

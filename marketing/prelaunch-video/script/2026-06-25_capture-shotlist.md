# Bezier 実機キャプチャ・ショットリスト v2

> あなた（CEO）が **本物の Bezier** を録画するための台本。各クリップは絵コンテ v2 のカットに対応。
> ハイブリッド構成なので、ここで撮るのは **アプリ内の実体験**。HERO/CLOSE テロップ・eyebrow・CTA は後で Remotion で載せる。
> 狙いは「**あなたが指揮し、エージェントが並列で作り、あなたは動く画面でレビューする**」が伝わること。

---

## 0. 収録セットアップ（全クリップ共通）
- **解像度/フレーム**: 最低 1920×1080（できれば 2× Retina）/ **60fps**（スローに耐える）。書き出しは後で 1080p。
- **アプリ幅**: ウィンドウ固定（録画中にリサイズしない）。メニューバー自動非表示・OS 通知 OFF（Bezier 自身の通知は見せたいので §CLIP3 参照）。
- **テーマ**: **ライト**基本（モノクロ・余白が映える）。ダークなら全クリップ統一。
- **カーソル**: やや大きめ＋クリックエフェクト ON（Screen Studio 推奨＝自動ズーム＋滑らかカーソルでベジェ的に見える。無ければ QuickTime）。
- **データ**: デモ用の **実プロダクト**を1つ（あなたの既存 repo が理想＝“既存文脈から作る”が映える）。生々しいダミーデータで。
- **ペース**: 各操作はゆっくり一定。1操作したら1拍止める（編集で詰められる）。
- **絶対 NG**: 黒いターミナル/緑文字ログを主役に映す。長い読み込みスピナー。コード差分の生画面を“レビューの主役”として映す（←本動画はコードを見せない方針）。

## CLIP 1 → S1（Direct）｜目安 20–25s 録る（使うのは ~13s）
**狙い**: 普通の言葉で意図を渡すと、モックでなく **動くプレビュー**が返る。
1. 静かな状態から開始（2–3秒の“間”＝S0 用の余白）。
2. chat に意図を**一行**ゆっくりタイプ: 例 `Add an empty state to the dashboard when there are no projects.`
3. 送信 → エージェントが拾い → **動く（クリックできる）プレビュー**が立ち上がる瞬間を収める。
- **収めるべき**: 「言葉 → 動くもの」。プレビューが**本物に見える**こと（既存プロダクトの文脈）。
- **避ける**: コードエディタ/ターミナルを主役にする。

## CLIP 2 → S2（Review the running screen）｜目安 25–30s 録る（使うのは ~15s）★最重要
**狙い**: 差分を読まず、**動いてる画面をクリック＋注釈**してレビュー＝それが変更依頼。
1. 立ち上がったプレビューを少し**クリックして触る**（動いてることを示す）。
2. ズレてる要素に **pin** → 短く依頼（例 `make this CTA larger`）。
3. **pen** で線/囲い → 依頼文。**box** で領域 → 依頼文（pin/pen/box を別々にも撮る＝編集で1–2個に絞る）。
4. エージェントが直して **画面が更新**される様子を収める。
- **収めるべき**: 「画面を触る → 印 → 画面が直る」がコードを見せずに完結。
- **避ける**: ここで diff ビューを長く映さない（本動画はレビュー＝注釈で見せる）。

## CLIP 3 → S3（Parallel / Board）｜目安 30–40s 録る（使うのは ~17s）★orchestrator
**狙い**: 複数案を**同時に**走らせ、**一画面（Board）で俯瞰**、**ready で通知**、**1つ選ぶ**、**隔離で安全**。
1. **Product Board**（タイル＋状態）を開き、**複数の案/issue が同時に走っている**状態を映す（N-max 上限内で2–3枚）。
2. それぞれが別々の live preview として**並んでいる**こと（互いに干渉しない＝隔離）を見せる。
3. 1つが仕上がって **デスクトップ通知**（"ready"）が出る瞬間を収める（このクリップだけ Bezier 通知は ON）。
4. ボード上で見比べ、**気に入った1つを選ぶ/開く**。
- **収めるべき**: 「同時に複数」「一画面で全部見える」「通知」「選ぶ」。＝本物の orchestrator。
- **段取りのコツ**: 先に2–3案を start しておき、録画は「すでに複数走っている所」から始めると映える。同時起動は **N-max プレビュー上限**（DEC-040）内で。
- **補足素材**: ボードをゆっくりパン/ズームしただけのクリップも撮る（つなぎ・S0 背景用）。

## CLIP 4 → S4（Ship / hand off）｜目安 25–30s 録る（使うのは ~9s）
**狙い**: クリックできる共有リンク／clean PR、main は汚さない。
1. 選んだ案を **Share** → **リンク生成**（コピーされる様子）。
2. （あれば）**スマホ等で開いて触る**絵＝“誰でもクリックできる動く体験”が強い。
3. **clean PR** を開く（隔離 worktree/branch から、main を触らずに）。アプリ内 PR オープン表示 or GitHub PR 画面。
- **収めるべき**: 「1リンクで共有」「engineer には clean PR」「main 無汚染」。

# Bezier Pre-launch — スクリプト v2（EN master / JP）

> **背骨**: 自己像シフト（つくる人 → 導く人）。HERO と CLOSE で立てる。
> **方針**: pre-launch はまず **テロップのみ（captions-only）で成立**させる版を正とする（声優・収録不要で即出せる）。
> 各カットに任意の **VO 行**も併記。日本語は字幕トラック or 日本語版テロップ。
> ブランド名 **Bezier** は原語。**LP コピーは引用しない**。名前由来の比喩語（handle/curve/control point）は使わない。
> 文体は orchestrator ジャンル踏襲＝**短い断定＋並列語＋制御語**。働きの単位＝diff/PR でなく **画面/フロー**、レビュー＝読む/merge でなく **クリック/注釈**。

凡例: **CAPTION** = 画面テロップ（必須）/ *VO* = 任意ナレーション / `micro` = 小さい補助テロップ

- **eyebrow (EN)**: `THE AGENT ORCHESTRATOR — FOR DESIGNERS`

- **CAPTION (EN)**: `Describe what you want.` → `An agent builds it — running, not a mockup.`

### S2 — Review the running screen（0:22–0:37）★差別化
- **CAPTION (EN)**: `Review the running screen — not the diff.`
- **CAPTION (JP)**: `差分じゃなく、動く画面でレビュー。`

- *VO (JP)*: 「差分は読まない。動いてる画面をクリックして、ズレてる所に印をつける——その印が、そのまま変更依頼。」

### S3 — In parallel / the Board（0:37–0:54）★orchestrator
- **CAPTION (EN)**: `Run several directions at once.` → `Each one isolated. You're pinged when one's ready.` → `Keep the one that feels right.`

- *VO (JP)*: 「複数の案を同時に走らせる——各々が専用プレビューで隔離。仕上がったら通知。気に入った1つを残す。」

### S4 — Ship / hand off（0:54–1:03）
- **CAPTION (EN)**: `Ship a link anyone can click.` → `Hand engineers a clean PR.`
